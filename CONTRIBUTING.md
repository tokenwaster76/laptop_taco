# Contributing to Laptop Taco

Thanks for being here. Two rules:

1. **Make it funny.**
2. **Keep it tiny.**

If your PR needs a database, Kubernetes, or a pitch deck, it is not a taco.

## What we want

- Bug fixes in `taco` (the script).
- Copy polish in `README.md` or the launch drafts under `marketing/posts/`.
- Small, single-purpose features that earn their lines.
- More edge-case coverage in `.github/workflows/macos-smoke-test.yml`.
- Better ASCII art.

## What we don't want

- A configuration file.
- A daemon.
- A menu-bar app.
- A Rust rewrite (your Rust rewrite is welcome at a different URL).
- Linux or Windows "support" — `caffeinate` is the dependency on purpose.
- Sentry / PostHog / any telemetry.
- A `--quiet` flag whose only purpose is to delete the safety warning. The safety warning stays.

## Development

```bash
git clone https://github.com/tokenwaster76/laptop_taco.git
cd laptop_taco
chmod +x taco

# syntax check
bash -n taco

# basic sanity (works on any OS — flags run before the Darwin guard)
./taco --help
./taco --version

# diagnose your local macOS setup
./taco doctor
```

On macOS, run the happy path:

```bash
./taco "echo cooking && sleep 1 && echo done"
./taco "exit 42"; echo "got $?"
```

The repo's `macOS smoke test` workflow runs on every push and PR — it boots `macos-latest`, syntax-checks the script, and exercises the public surface (`--help`, `--version`, `doctor`, happy path, exit codes 42 and 127, shell features, embedded quotes, the missing-command guard).

## Style

- Bash, not Zsh-isms. Target Bash 3.2 (the version shipped with macOS).
- `set -u` and `set -o pipefail`, **not** `set -e` — the script needs to capture the child's exit code from `wait` precisely.
- One file. The whole thing lives in `taco`.
- No new dependencies. `caffeinate`, `pmset`, `osascript`, `date`, `kill`, `wait`, `trap` are the budget.
- Quote your variables. `"$pid"`, not `$pid`.
- Comments only when WHY isn't obvious. Don't restate what the code already says.

## Marketing kit changes

If your PR touches `marketing/`:

- Run `marketing/scripts/audit-tree.sh` and confirm clean.
- Every API poster must keep its env-less fail-clean exit path (1 with explicit `export` instructions).
- All `curl` calls keep `--max-time 60` (or document why not).
- DEV.to and Hashnode stay hard-coded `published: false`.
- The Reddit script keeps its `REDDIT_AUTO_POST_OPT_IN=true` gate.
- Auto-announce workflow keeps the `release.published` trigger (no cron).

## Authorship

Commit as yourself, not as an AI assistant. No "Generated by", no "Co-authored-by: Claude/ChatGPT/etc.", no AI-attribution comments anywhere. See `SECURITY.md` and `marketing/MARKETING-IS-PUBLIC.md` for the wider policy.

## Pull request flow

1. Open against `main`.
2. Make sure the macOS smoke test is green.
3. Use the PR template — it asks for what changed, why, and how you verified.
4. Be ready for "this isn't taco-shaped" pushback if your PR is large. Tiny is the point.
